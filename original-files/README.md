# Мероприятия для организованных групп в Библиотеке книжных героев
[Открыть vbkg.pr-cbs.ru](https://vbkg.pr-cbs.ru "Мероприятия для организованных групп в Библиотеке книжных героев")

убрали доступ к поисковым системам.
теперь БД хранится в папке \server


в .env
```
EMAIL_USER= ***почта, куда отправлять письмо*** 
EMAIL_PASS= ***пароль приложения (генериться в почтовом сервисе)***
ADMIN_EMAIL= ***почта админа, откуда шлётся письмо***
FRONTEND_URL=http://localhost:3000 (используется в ***app.use(cors)*** и в ***const PORT = process.env.PORT || 3000;***  на сервере.
```

### server.js

В файле server.js строчка ```const adapter = new FileSync('db.json');``` 

должна поменяться на такую: 

```const adapter = new FileSync('./data/db.json');```

В этом же файле после строчки 
```app.use(express.static(path.join(__dirname, '../public')));```

добавляется строчка:
```app.set('trust proxy',true);```

Если меняется почтовый сервис (гугл, яндекс и тд), то в ```const emailTransporter = nodemailer.createTransport``` сервера нужно изменить имя сервиса

Сервер автоматически бронирует мероприятие, чтобы не случилось, что мероприятие забронировали за 5 минут до начала. В 11 и 12 строке server.js меняется периодичность проверки автоматического бронирования. В 13 строке - периодичность создания бэкапа

```const checktime```

```const AUTO_BOOK_DAYS```

```const backtime```

Сервер проверяет количество созданных мероприятий и удаляет лишние, чтобы не перегружать БД и клиента (т.к. клиент получает всю БД). Сейчас проверка стоит каждый месяц: если больше 1200 мероприятий, удаляет 300 (и связанные бронирования). При этом делается бэкап БД. Должно хватить на пару-тройку лет (из расчета, что каждую неделю проходит 10 мероприятий - на самом деле меньше)...

### const CLEANUP_CONFIG

При использовании старой БД автоматом проставляется, что мероприятие платное (500 руб).

При удалении картинки, она удаляется из БД, но остаётся в ```server\uploads``` в ```app.delete('/api/events/')``` можно настроить удаление файла из папки.
